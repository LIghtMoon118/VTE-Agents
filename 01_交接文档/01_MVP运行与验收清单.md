# MVP 运行与验收清单

## 1. 运行环境

最低要求：

- macOS 或 Linux。
- Node.js 18 及以上。
- 浏览器：Chrome、Edge 或 Safari。

当前源码位置：

```text
02_MVP源码/vte-agent-mvp/
```

## 2. 本地启动

进入源码目录：

```bash
cd "02_MVP源码/vte-agent-mvp"
```

检查语法：

```bash
npm run check
```

启动服务：

```bash
npm start
```

浏览器访问：

```text
http://localhost:8787
```

也可双击源码目录下的：

- `start-mvp.command`
- `start-and-open-mvp.command`

## 3. 基础验收

研发团队拿到包后，先做以下验收。

| 验收项 | 操作 | 通过标准 |
|---|---|---|
| 服务启动 | 执行 `npm start` | 8787 端口可访问 |
| 页面加载 | 打开 `http://localhost:8787` | 首页和各页签正常显示 |
| 单病例分析 | 输入病例文本并点击分析 | 生成风险等级、证据、任务和综合结论 |
| 病历审查 | 分析后查看病历审查页 | 出现病历证据链、薄弱点和补强动作 |
| 多模态导入 | 上传 docx/pdf/图片/文本样例 | 能抽取文字并触发分析，失败时有明确提示 |
| 知识图谱 | 分析病例后切到知识图谱 | 当前病例中心节点和相关子图更新 |
| RAG 问答 | 查看 RAG 页或提问 | 返回证据条目和来源说明 |
| 报告导出 | 点击 Markdown/HTML 导出 | 生成报告并含证据链章节 |
| 模型回退 | 不配置模型时运行 | 显示本地规则/RAG分析，不伪装成大模型 |

## 4. 建议测试病例

### 场景 A：围术期高危

```text
72岁男性，腹盆腔肿瘤术后卧床3天，D-二聚体升高，无活动性出血，血小板正常，肌酐正常。需评估VTE风险和围术期预防策略。
```

期望：高危；提示围术期、肿瘤、卧床、D-二聚体；不应提示妊娠。

### 场景 B：疑似 PE 急症线索

```text
65岁男性，术后第5天突发胸痛、呼吸困难，D-二聚体升高，血氧下降，当前无活动性出血。需评估肺栓塞风险和下一步处理。
```

期望：高危或急症优先；提示疑似 PE、影像/会诊/抗凝安全核查；不应泛化为普通预防建议。

### 场景 C：病历缺口明显

```text
80岁女性，髋部骨折术后，长期卧床，病程中提到下肢肿胀。病历未见VTE风险评估记录，未见出血风险评估，未见机械预防执行记录，出院记录未见抗凝和随访安排。
```

期望：指出病历证据链薄弱点；给出需要补齐的评估、医嘱、护理、宣教和随访记录。

## 5. 不通过时优先排查

1. Node.js 版本是否低于 18。
2. 8787 端口是否被占用。
3. `server.js` 或 `public/app.js` 是否有语法错误。
4. 上传文件是否超过当前 28MB 限制。
5. 扫描 PDF/图片是否过于模糊，导致 OCR 无法识别。
6. 是否误以为未配置外部模型时也会调用大模型；默认应回退到本地规则/RAG。
