# 工程化改造任务书

## 1. 总目标

把当前本地 MVP 改造成可在院内测试环境部署、可接入脱敏病例、可被临床团队验证的一期工程原型。

工程化不等于重写一个聊天机器人。研发目标应围绕 VTE 专病工作流：病例输入、风险识别、证据检索、病历审查、路径质控、任务生成、人工确认、审计留痕和报告导出。

## 2. 推荐架构

### 后端分层

建议拆分为：

- `routes/`：API 路由。
- `services/analysis/`：病例分析、风险识别、急症线索。
- `services/emr-review/`：病历审查、证据链、补强清单。
- `services/rag/`：知识库检索、证据排序、引用。
- `services/graph/`：知识图谱生成与病例激活。
- `services/document-ingest/`：PDF/Word/图片/OCR。
- `services/model/`：模型适配、本地模型、OpenAI-compatible 接口。
- `services/audit/`：审计编号、日志、人工确认记录。
- `config/`：规则、模型、端口、数据源配置。
- `tests/`：单元测试、接口测试、病例回归测试。

### 数据层

一期可选：

- SQLite：适合单机和内网测试。
- PostgreSQL：适合多人联调和后续正式部署。

建议至少存储：

- 脱敏患者/就诊 token。
- 输入文本或结构化字段摘要。
- 分析结果 JSON。
- 证据引用。
- 审计编号。
- 人工确认状态。
- 版本号。
- 操作时间和操作者。

### RAG 层

建议升级为：

1. 文献导入。
2. 文献切片。
3. 元数据标注：指南/共识/综述/研究、年份、适用人群、证据等级。
4. 向量化。
5. 关键词 + 向量混合检索。
6. 重排序。
7. 输出引用。
8. 专家审核状态。
9. 版本管理。

## 3. 任务拆分

### A. 后端服务化

交付物：

- 拆分后的后端工程目录。
- API 文档。
- 环境变量说明。
- 测试启动脚本。
- 统一错误码。

验收：

- 原有核心 API 行为不丢失。
- `npm run check` 或等价检查通过。
- 至少 10 个病例回归测试通过。

### B. 病历解析与结构化抽取

交付物：

- PDF/Word/图片/文本解析服务。
- OCR 失败和低置信度提示。
- 结构化字段抽取：诊断、手术、检验、医嘱、护理、影像摘要、时间点。

验收：

- 清晰 docx、文本型 PDF、扫描 PDF、图片样例均可处理。
- 无法识别时不静默失败。
- 结果能进入 `/api/analyze` 和 `/api/emr/review`。

### C. 临床规则与推理链

交付物：

- 可配置规则库。
- 规则版本号。
- 规则命中解释。
- 专家审核记录。

验收：

- 非妊娠病例不得出现妊娠建议。
- 无出血证据不得泛化输出“高出血风险”。
- 无 PE 症状不得点亮 PE 急症路径。
- 急症线索病例必须优先提示急症处理。

### D. RAG 与知识库

交付物：

- 文献切片库。
- 向量检索服务。
- 证据等级排序。
- 来源引用和版本记录。

验收：

- 每条关键建议至少能追溯到规则或证据来源。
- 指南/共识优先于普通研究。
- 检索结果能按 DVT、PE、围术期、妊娠、肿瘤、出血等场景区分。

### E. 前端产品化

交付物：

- 更清晰的启动首页。
- 病例导入状态闭环。
- 证据链跨页同步。
- 报告导出和打印。
- 医生端/护士端/管理端视图雏形。

验收：

- 用户导入病例后能明确看到“已识别什么、判断什么、缺什么、下一步做什么”。
- 页面不出现明显套话和无关建议。
- 高危颜色、图标、标题等界面元素一致。

### F. 部署与安全

交付物：

- 内网测试部署方案。
- 服务启动脚本。
- 日志路径。
- 数据库初始化脚本。
- 权限和审计方案。
- 备份和回滚方案。

验收：

- 可在测试服务器稳定启动。
- 有健康检查接口。
- 有访问日志和错误日志。
- 有基本鉴权或网关接入方案。

## 4. 不建议研发团队做的事

- 不要把现有 MVP 直接改成通用聊天机器人。
- 不要在没有院内授权的情况下接入真实病历。
- 不要让系统自动生成最终医嘱。
- 不要用单一大模型输出替代规则、证据和人工确认。
- 不要把演示病例结果包装成临床验证结果。

## 5. 第一阶段完成定义

两到四周内，第一阶段应达到：

- 可在研发本地和院内测试环境稳定运行。
- 可导入脱敏病历文件并完成全链路分析。
- 可用结构化 `patientContext` 调用。
- 至少 30-50 例脱敏病例完成回归测试。
- 形成一份《VTE智能体一期临床验证问题清单》。
- 形成一份《下一阶段数据接入与院内试点方案》。
