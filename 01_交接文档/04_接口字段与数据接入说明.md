# 接口字段与数据接入说明

## 1. 当前接口

当前 MVP 主要接口见：

```text
02_MVP源码/vte-agent-mvp/docs/API.md
```

重点接口：

| 接口 | 用途 |
|---|---|
| `GET /api/health` | 健康检查 |
| `GET /api/demo-cases` | 获取模拟病例 |
| `POST /api/analyze` | 单病例分析 |
| `POST /api/emr/review` | 病历审查和证据链分析 |
| `POST /api/document/ingest-preview` | 多模态病历导入预检 |
| `POST /api/knowledge/graph/case` | 当前病例知识图谱激活 |
| `POST /api/batch-analyze` | 批量病例分析演示 |
| `POST /api/report/markdown` | Markdown 报告导出 |
| `POST /api/report/html` | HTML 报告导出 |

## 2. 推荐输入结构

研发团队应优先使用 `patientContext`，不要长期只依赖大段自由文本。

```json
{
  "patientContext": {
    "patientToken": "demo-patient-001",
    "encounterToken": "demo-encounter-001",
    "department": "血管外科",
    "admissionTime": "2026-07-11T08:00:00+08:00",
    "age": 72,
    "sex": "男",
    "diagnoses": ["腹盆腔肿瘤", "术后状态"],
    "procedures": ["腹盆腔肿瘤手术"],
    "labs": [
      {
        "name": "D-二聚体",
        "value": "升高",
        "unit": "",
        "time": "2026-07-11T10:00:00+08:00"
      }
    ],
    "orders": ["机械预防", "下肢静脉彩超"],
    "nursingRecords": ["卧床3天", "当前无活动性出血"],

```

## 3. 一期最低字段

P0 字段：

- `patientToken`
- `encounterToken`
- `department`
- `age`
- `sex`
- `diagnoses`
- `procedures`
- `labs`
- `orders`
- `nursingRecords`
- `freeText`

P1 字段：

- `admissionTime`
- 检验时间。
- 手术时间。
- 影像/超声报告摘要。
- 出院计划。
- 抗凝用药起止时间。

## 4. 推荐数据来源

| 来源 | 输入内容 |
|---|---|
| HIS/住院系统 | 患者、就诊、科室、入院时间 |
| EMR | 诊断、病程、入院记录、出院计划 |
| 手麻系统 | 手术名称、术式、手术时间 |
| LIS | D-二聚体、血小板、Hb、PT/APTT、肌酐 |
| 医嘱系统 | 抗凝、机械预防、影像检查、复查医嘱 |
| 护理系统 | 卧床、活动、机械预防、宣教、复评 |
| PACS/超声 | 下肢静脉彩超、CTPA、CTA/CTV 报告摘要 |

## 5. 推荐输出与回写

| 输出 | 推荐去向 | 说明 |
|---|---|---|
| `riskLevel` | 医生端、管理端 | 风险等级 |
| `riskFactors[]` | 医生端 | 风险解释 |
| `bleedingFlags[]` | 医生端 | 抗凝安全核查 |
| `urgentFlags[]` | 医生端、管理端 | 急症线索 |
| `recommendations[]` | 医生端 | 建议草案，必须医生确认 |
| `reviewItems[]` | 病历审查端 | 病历缺口和流程核查 |
| `roleTasks.nurse[]` | 护理端 | 护理任务和复评提醒 |
| `patientEducation[]` | 护理端/患者端 | 宣教内容 |
| `qualityIndicators[]` | 管理端 | 质控指标 |
| `audit.auditId` | 全系统 | 审计追溯 |

## 6. 联调顺序

### 阶段 1：离线脱敏病例

只用脱敏文本和文件，跑通输出逻辑，不回写医院系统。

### 阶段 2：结构化字段联调

用 `patientContext` 接口跑通字段输入，验证输出字段是否可用于医生端、护士端和管理端。

### 阶段 3：测试环境流程节点

在测试环境接入入院、术前、术后、病情变化、出院前等触发节点。

### 阶段 4：人工确认与审计

增加医生确认、护理确认、管理查看和审计日志。

## 7. 必须提前确认的问题

1. 哪些字段可以通过接口实时获取，哪些只能批量导出。
2. 脱敏 token 由谁生成。
3. 是否允许测试环境保存脱敏病历摘要。
4. 医生确认后的结果回写到哪里。
5. 审计日志由智能体保存，还是由院内平台统一保存。
6. 是否需要通过院内统一网关和统一身份认证。
7. OCR/文件解析服务部署在何处。
