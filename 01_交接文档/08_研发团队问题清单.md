# 研发团队问题清单

## 1. 启动前必须回答

1. 研发团队是否只做 MVP 工程化，还是同时负责院内系统集成。
2. 是否有院内测试服务器。
3. 是否允许使用脱敏真实病例。
4. 是否有统一身份认证和网关要求。
5. 是否已有院内私有模型或本地模型环境。
6. 是否需要兼容既有 VTE 管理系统。

## 2. 数据问题

1. HIS/EMR/LIS/护理/医嘱系统分别由谁提供接口。
2. 字段是实时接口、数据库视图、批量导出，还是人工上传。
3. 患者和就诊脱敏 token 由哪一侧生成。
4. 检验名称是否有统一字典。
5. 护理记录能否结构化提供。
6. 影像报告是完整报告、摘要，还是只提供检查结论。

## 3. 临床问题

1. 一期优先覆盖 DVT、PE、围术期、肿瘤、妊娠/产褥中的哪些场景。
2. 院内 VTE 防控路径和量表以哪一版为准。
3. 哪些输出必须医生确认。
4. 护士端最需要哪些任务提醒。
5. 管理端最需要哪些质控指标。
6. 出院随访是否纳入一期。

## 4. 模型与知识库问题

1. 是否建设正式向量库。
2. 文献和指南由谁审核。
3. 知识库更新频率如何定。
4. 大模型只做解释生成，还是参与风险判断。
5. 模型输出如何记录证据来源。
6. 如何处理模型不确定、证据冲突和低置信度。

## 5. 合规问题

1. 是否需要伦理审批或质控项目备案。
2. 是否进入医疗器械软件评估范围。
3. 外部企业能否接触数据。
4. 日志保存多久。
5. 报告是否进入正式病历。
6. 对外展示截图是否需要脱敏和审批。

## 6. 第一轮研发会建议结论

第一轮会议不要只讨论界面。建议必须形成以下结论：

- 一期目标。
- 一期数据范围。
- 一期部署环境。
- 一期接口字段。
- 一期临床验证病例数。
- 一期不做事项。
- 两周内交付物。
