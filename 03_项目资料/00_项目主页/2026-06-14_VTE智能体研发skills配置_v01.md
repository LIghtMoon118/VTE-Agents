# VTE 智能体研发 skills 配置 v01

建立日期：2026-06-14

## 一、已新增安装

本次为 VTE 智能体后续思维探讨、创意分析和研发推进，新增安装以下 skills：

1. `jupyter-notebook`
   - 用途：文献矩阵分析、知识图谱探索、批量病例变量分析、统计和可视化原型。
   - 适合任务：PubMed/CNKI 文献分层、主题聚类、知识实体抽取、病例队列初步分析。

2. `playwright`
   - 用途：自动化打开和测试 VTE 智能体网页，检查前端交互、截图、演示稳定性。
   - 适合任务：企业演示前验证、页面回归测试、不同屏幕尺寸检查。

3. `security-threat-model`
   - 用途：建立内网服务器、病历数据、企业接口、模型和知识库的威胁模型。
   - 适合任务：数据不出院设计、权限边界、攻击面分析、企业合作安全审查。

4. `security-best-practices`
   - 用途：将安全边界转化为研发规范和部署要求。
   - 适合任务：接口鉴权、日志审计、脱敏导出、内网部署、密钥和权限管理。

## 二、当前已有且应优先使用的能力

1. `pubmed-database`
   - 用于 PubMed 检索式设计、E-utilities 批量抓取、PMID/摘要/MeSH 元数据整理。

2. `paper-lookup` / `paper-2-web` / `literature-review`
   - 用于文献检索、综述框架、证据分层和重点文献阅读。

3. `pdf`
   - 用于 PDF 全文抽取、切片、表格和文献证据处理。

4. `networkx`
   - 用于 VTE 知识图谱原型、实体关系网络和图谱可视化。

5. `clinical-decision-support`
   - 用于把 VTE 智能体输出约束为临床辅助决策，而非自动诊疗。

6. `clinical-reports`
   - 用于病历深度分析、病例复盘和医疗质量审查报告。

7. `citation-management`
   - 用于文献引用、证据来源和论文/标书参考文献管理。

8. `exploratory-data-analysis` / `statistical-analysis`
   - 用于文献矩阵、病例队列和真实世界研究初步分析。

## 三、建议使用方式

### 1. 文献知识库和知识图谱

优先组合：

- `pubmed-database`
- `pdf`
- `literature-review`
- `networkx`
- `jupyter-notebook`

目标：

- 扩展 PubMed/CNKI 文献矩阵。
- 抽取指南、综述和高质量研究。
- 构建 VTE 实体和关系。
- 形成 RAG + 知识图谱混合知识底座。

### 2. 病历深度分析和临床辅助决策

优先组合：

- `clinical-decision-support`
- `clinical-reports`
- `statistical-analysis`

目标：

- 分析单份电子病历。
- 提示 VTE 风险、出血风险和流程缺陷。
- 输出医生确认前的建议草案。
- 生成医疗质量和病历完善建议。

### 3. 产品原型和企业演示

优先组合：

- `playwright`
- `jupyter-notebook`

目标：

- 验证网页智能体可用性。
- 自动截图。
- 展示知识库检索、病例分析和接口能力。

### 4. 内网部署和企业合作安全边界

优先组合：

- `security-threat-model`
- `security-best-practices`

目标：

- 设计数据不出院架构。
- 明确企业不能接触的资产。
- 设计接口鉴权、日志审计、导出分级。
- 支撑伦理、信息安全和横向课题协议。

## 四、注意事项

新安装 skills 需要重启 Codex 后才能在后续会话中被自动识别和触发。当前会话仍可继续推进，但正式使用这些新增 skills 的完整能力，建议重启后继续。
