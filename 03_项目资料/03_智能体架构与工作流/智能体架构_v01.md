# VTE 智能体架构 v01

建立日期：2026-06-12

## 一、总体架构

临床数据层 → 风险评估层 → 知识与规则层 → 智能体任务层 → 人工确认层 → 临床执行层 → 反馈评价层。

## 二、智能体角色

### 1. 风险评估智能体

- 汇总患者风险因素。
- 识别风险变化。
- 提示复评时间。
- 解释主要风险来源。

### 2. 防控建议智能体

- 检索指南和院内路径。
- 结合出血风险、禁忌证和患者状态生成建议草案。
- 标记需医生确认的关键建议。

### 3. 护理任务智能体

- 生成机械预防、活动指导、健康宣教和复评提醒。
- 跟踪任务执行状态。
- 提示漏评漏防。

### 4. 患者随访智能体

- 生成出院教育内容。
- 提醒抗凝、活动和复诊。
- 收集症状和依从性信息。
- 对警示症状触发人工处理。

### 5. 质控管理智能体

- 汇总高危患者。
- 监测评估率、预防率、处理率和随访率。
- 生成科室或病区反馈。

### 6. 科研评价智能体

- 整理研究数据。
- 生成变量缺失和结局核查清单。
- 支持模型评价和真实世界研究。

## 三、人机协同原则

- 智能体生成建议，医生确认关键决策。
- 护理任务需与实际工作流对齐。
- 患者端内容以教育和提醒为主，不替代诊疗。
- 所有关键输出留痕，可追溯来源和版本。

## 四、最小可行工作流

1. 患者入院或术前触发 VTE 风险评估。
2. 智能体汇总风险因素并生成解释。
3. 根据院内路径生成防控建议草案。
4. 医生确认或调整。
5. 护士端生成执行任务和复评提醒。
6. 出院时生成患者教育和随访计划。
7. 管理端汇总质控指标。
