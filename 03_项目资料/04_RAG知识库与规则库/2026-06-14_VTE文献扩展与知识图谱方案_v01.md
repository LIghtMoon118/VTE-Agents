# VTE 文献扩展与知识图谱方案 v01

建立日期：2026-06-14

## 一、总体策略

VTE 知识库不应一次性无差别吞入全部文献。PubMed 2006-2026 年题名核心命中文献约 3.8 万篇，题名/摘要扩展命中约 8.0 万篇，MeSH 命中约 6.3 万篇。若全部进入 RAG，会造成噪声、重复和检索不稳定。

建议采用三层知识库：

1. 核心证据库：指南、共识、系统综述、Meta、RCT、高质量队列、重要真实世界研究。
2. 扩展文献库：题名/摘要命中的 DVT、PE、VTE、静脉血栓形成、肺栓塞等相关文献。
3. 全文证据库：已获取 PDF 的文献全文切片，逐步向量化。

## 二、英文检索策略

### 1. 核心题名检索

用于构建高相关文献库：

```text
"venous thromboembolism"[ti]
"deep vein thrombosis"[ti]
"deep venous thrombosis"[ti]
"pulmonary embolism"[ti]
"venous thrombosis"[ti]
"pulmonary artery embolism"[ti]
```

### 2. 扩展题名/摘要检索

用于扩大覆盖面：

```text
"venous thromboembolism"[tiab]
"deep vein thrombosis"[tiab]
"deep venous thrombosis"[tiab]
"pulmonary embolism"[tiab]
"venous thrombosis"[tiab]
"pulmonary artery embolism"[tiab]
```

### 3. MeSH 检索

用于补充规范主题词：

```text
"Venous Thromboembolism"[mh]
"Pulmonary Embolism"[mh]
"Venous Thrombosis"[mh]
```

### 4. 主题过滤

建议按以下主题分层：

- 指南/综述/Meta。
- AI/机器学习/预测模型。
- 影像/CTPA/超声诊断。
- 预防/护理/康复。
- 抗凝/出血/药物安全。
- 介入/PERT/血栓清除/滤器。
- 真实世界/队列/流行病学。

### 3. CNKI 执行方式

CNKI 不建议直接用非授权爬虫。推荐方式：

- 在 CNKI 页面按题名/主题检索导出 CSV/Excel。
- 记录检索式、日期、数据库范围和筛选条件。
- 导入本地 `VTE文献矩阵`。
- 再由智能体导入结构化知识库。

## 六、知识图谱关系设计

| 关系 | 示例 |
|---|---|
| 增加风险 | 肿瘤 → 增加风险 → VTE |
| 用于评估 | Caprini → 用于评估 → 围术期 VTE 风险 |
| 用于诊断 | CTPA → 用于诊断 → PE |
| 用于预防 | 机械预防 → 用于预防 → DVT |
| 存在禁忌 | 活动性出血 → 限制使用 → 药物抗凝 |
| 触发复评 | 手术后 → 触发 → VTE 复评 |
| 需要确认 | 药物预防建议 → 需要 → 医生确认 |
| 支持证据 | 文献 → 支持 → 某条推荐 |

## 七、研发路线

1. 继续完善结构化文献矩阵。
2. 把高优先级 PDF 全文抽取成文本。
3. 切片并建立向量索引。
4. 从指南/综述中抽取知识图谱三元组。
5. 将图谱实体与病历变量字典对齐。
6. 在病例分析中同时返回 RAG 证据和图谱解释路径。
