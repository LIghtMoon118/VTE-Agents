# 企业沟通前产品成品化与专利预布局冲刺方案

日期：2026-06-16  
用途：在后续企业沟通前，尽量由临床团队先完成可演示原型、技术边界、知识产权预布局和对外披露边界，避免把核心想法过早、无保护地交给企业。

## 一、基本判断

当前 VTE 智能体主要由临床团队自主推进。后续企业可以参与工程化、部署、接口、运维和产品化，但在正式沟通前，应先完成四件事：

1. **产品上可展示**：不只是 PPT，而是能运行、能输入病例、能输出分析、能展示 RAG、图谱、接口和驾驶舱。
2. **技术上可界定**：明确哪些是智能体核心能力，哪些是普通 VTE 管理系统功能，哪些只是行业通用需求。
3. **知识产权上可主张**：先形成发明点清单、交底书草案、软著材料和版本证据。
4. **合作上可控边界**：企业沟通前不披露权利要求级别细节，不交付真实数据，不无条件开放核心规则和知识库。

## 二、专利与披露的基本边界

根据国家知识产权局公开信息，专利法中的发明创造包括发明、实用新型和外观设计。发明通常指对产品、方法或其改进提出的新的技术方案；实用新型主要涉及产品形状、构造或其结合；外观设计涉及产品整体或局部外观设计。

对本项目而言，主要考虑：

- **发明专利**：保护 VTE 智能体的方法、系统、流程、知识图谱/RAG 协同、病例分析和质控闭环。
- **软件著作权**：保护具体软件系统、代码、界面组织和文档。
- **外观设计**：若后续形成独特的可视化界面、知识图谱展示或驾驶舱界面，可评估是否适合另行保护。

注意：公开披露可能影响新颖性。虽然法律上存在少数“不丧失新颖性”的例外情形，但不能把普通企业沟通、路演或无保密协议演示当作安全公开。企业沟通前建议至少完成发明交底书和保密版/公开版演示材料分级。

## 三、企业沟通前的三层保护

### 1. 产品保护

先把当前 MVP 打磨到能稳定演示：

- 单病例文本分析。
- PDF/Word/图片病历导入预检。
- VTE 风险、出血风险、急症线索识别。
- RAG 知识库问答。
- 病例相关知识图谱。
- 病历深度审查。
- 医生/护士/患者/管理者/科研端任务生成。
- 院内队列筛选和批量分析模拟。
- 管理者驾驶舱。
- 模型适配层。
- REST API 对接接口。

目标：企业看到的是一个已成形的智能体产品雏形，而不是“希望企业帮我从零实现”。

### 2. 知识产权保护

企业沟通前至少准备：

- 发明点清单。
- 1-2 项优先发明专利交底书草案。
- 软件著作权登记材料草案。
- 项目版本记录。
- 演示系统截图和功能说明。
- 核心代码与文档时间戳证据。

目标：先证明核心创意、产品结构和技术路线由临床团队提出并先行实现。

### 3. 合作边界保护

企业沟通前准备两套材料：

1. **公开演示版**
   - 展示功能、流程、界面、产品定位。
   - 不展示核心规则细节、知识库结构、权利要求级技术方案。
   - 不输入真实患者数据。

2. **保密沟通版**
   - 仅在 NDA 或明确合作框架下展示。
   - 可讨论接口、部署、工程任务和验证路径。
   - 核心发明点、算法细节、规则库、知识图谱构建逻辑仍需分层披露。

## 四、建议优先布局的发明点

### 发明点 1：病例驱动的 VTE 智能体协同决策方法

核心思想：

将患者病历文本、结构化数据和多模态病历资料输入智能体，由智能体完成 VTE 风险识别、出血风险核查、急症线索判断、RAG 证据检索、病例知识图谱激活和多角色任务生成。

可能保护点：

- 病例输入到多角色任务输出的智能体工作流。
- 风险因素、急症线索、禁忌信号与证据检索的协同触发机制。
- 医生、护士、患者、管理者、科研端任务分发。
- 人工确认和审计日志机制。

### 发明点 2：基于病例相关知识图谱的 VTE 风险解释方法

核心思想：

根据当前病例直接激活 VTE 知识图谱中的相关节点，并按强/中/弱相关显示风险因素、急症线索、证据、诊断和处置路径；无关、否定或相反信息不显示。

可能保护点：

- 病例特征到知识图谱节点的激活方法。
- 相关度分层和可视化布局。
- 否定信息和正常值对图谱节点的抑制机制。
- RAG 证据与图谱节点联动。

### 发明点 3：妊娠、股青肿、疑似 PE 等特殊场景的急症优先 VTE 智能分析方法

核心思想：

普通 VTE 预防系统常以评分和防控建议为中心；本方法在识别疑似 PE、症状性 DVT、股青肿/肢体灌注受威胁等信号时，自动将普通预防后置，优先生成急症处理、影像检查、会诊和抗凝安全核查建议。

可能保护点：

- 急症线索优先级规则。
- 普通预防建议后置机制。
- 妊娠、肿瘤、围术期等特殊人群的路径约束。
- 急症路径与证据检索、任务生成、护理观察项联动。

### 发明点 4：面向院内批量病历的 VTE 智能分析与管理驾驶舱方法

核心思想：

在院内服务器对指定时间段、科室、诊断、手术或风险人群进行批量分析，生成高危患者、急症线索、科室分布、漏评漏防、护理执行、出院教育和科研变量缺失等统计结果。

可能保护点：

- 批量病历纳入和筛选机制。
- 单病例智能体分析结果向管理指标的汇总机制。
- 管理者驾驶舱指标生成方法。
- 原始病历不出院，仅导出脱敏统计和审计结果的流程。

### 发明点 5：VTE RAG 知识库与指南优先证据检索方法

核心思想：

将 VTE 指南、共识、SCI 和中文核心期刊论文构建为知识库，对病例相关问题进行检索时，优先返回国外权威指南、国内中华级指南/共识和最新证据，并结合病例相关度进行重排序。

可能保护点：

- VTE 证据等级和来源优先级排序。
- 病例主题和证据亚型匹配。
- 妊娠 VTE、DVT、PE、股青肿、出血风险等场景化检索策略。
- RAG 回答中知识库缺口提示机制。

## 五、优先申请策略

建议先准备 2 项发明专利交底书：

1. **VTE 专病智能体协同分析与多角色任务生成方法及系统**
   - 覆盖最宽。
   - 保护智能体总体工作流。
   - 适合企业沟通前优先占位。

2. **基于病例相关知识图谱和 RAG 的 VTE 风险解释与防控建议生成方法**
   - 保护差异化核心。
   - 覆盖病例图谱、相关性显示、指南优先证据和建议生成。

第三项可作为后续增强：

3. **面向院内批量病历的 VTE 智能分析、质控复盘和管理驾驶舱方法**
   - 与内网服务器和真实世界数据分析结合。
   - 适合后续横向课题或院内试点前布局。

## 六、软件著作权建议

建议同步准备软著材料：

1. **VTE 专病智能体系统**
2. **VTE 病历智能审查与质控复盘系统**
3. **VTE 知识库与病例知识图谱分析系统**
4. **VTE 管理者驾驶舱与批量病历分析系统**

软著更偏软件表达形式，不能替代发明专利，但可作为阶段性成果和企业沟通的权属证据。

## 七、产品成品化冲刺清单

企业沟通前，当前 MVP 建议补齐：

1. **演示稳定性**
   - 默认使用规则/RAG，避免本地大模型导致卡顿。
   - 演示病例预置 3-5 个：妊娠股青肿、老年肿瘤围术期、疑似 PE、出血禁忌冲突、普通中危复评。

2. **临床正确性**
   - 年龄、BMI、妊娠、肿瘤、PE、出血禁忌等规则继续细化。
   - 图谱只显示病例相关节点。
   - RAG 问题必须病例聚焦。

3. **功能完整度**
   - 多模态病历导入入口。
   - 队列筛选入口。
   - 批量分析入口。
   - 管理者驾驶舱。
   - 报告导出。
   - API 文档。

4. **视觉和体验**
   - 宋式美学风格继续统一。
   - 图谱居中、完整、不遮挡。
   - 不同功能区底色和视觉层级清晰。

5. **文档材料**
   - 产品说明。
   - API 说明。
   - 部署说明。
   - 企业演示脚本。
   - 专利交底书草案。
   - 保密披露清单。

## 八、对企业沟通的边界话术

可以说：

> 目前我们已经形成 VTE 专病智能体的本地可运行 MVP，并围绕病例分析、知识库/RAG、知识图谱、病历审查、批量分析和管理者驾驶舱形成了自有技术路线。希望企业参与的是工程化、私有化部署、系统接口、性能优化、运维和规模化产品化。

不建议说：

- 这些想法还只是设想，希望企业帮我从零实现。
- 核心规则和知识库可以先全部给企业看。
- 后续知识产权比例可以先口头谈。
- 真实病历可以拿出来给企业测试。

建议对企业提出：

- 先签 NDA。
- 明确既有成果归属。
- 明确新增成果归属和共同开发边界。
- 明确企业只接触其工作所需的最小资料。
- 明确真实数据只在院内授权环境处理。

## 九、下一步 7 天执行安排

### 第 1-2 天：产品原型打磨

- 固定 3-5 个高质量演示病例。
- 修正病例分析和图谱显示问题。
- 完成演示脚本。

### 第 3-4 天：专利交底书草案

- 完成 2 项发明专利交底书初稿。
- 提炼技术问题、技术方案、流程图、创新点和实施例。
- 收集当前 MVP 截图和版本记录作为证据。

### 第 5 天：软著和文档

- 整理软件名称、功能说明、模块结构。
- 准备软著材料目录。
- 输出 README、API、部署说明和演示说明。

### 第 6 天：企业沟通材料

- 准备公开演示版材料。
- 准备保密沟通版材料。
- 准备合作任务边界清单。

### 第 7 天：综合检查

- 本地部署复测。
- 演示流程彩排。
- 核对哪些内容可公开，哪些内容暂不披露。

## 十、需要用户补充的信息

为了正式写专利交底书，需要补充：

1. 既有 VTE 智能管理系统已授权或已申请专利的名称、申请号、权利人。
2. 本次新智能体是否以个人、医院、团队或未来公司名义布局知识产权。
3. 是否已有计划合作企业名称和沟通时间。
4. 是否准备先走医院成果转化流程。
5. 是否有希望重点保护的模块：知识图谱、RAG、批量病历分析、驾驶舱、病历审查或多角色任务。

## 十一、重要提醒

本文件是研发和成果转化策略建议，不替代专业专利代理人或律师意见。正式申请前，应由专利代理人进行检索、权利要求设计和法律审查。
