# VTE 智能体

建立日期：2026-06-12  
项目性质：既有 VTE 智能管理系统的智能体升级方向 / 医学 AI 成果转化项目  
当前状态：已建立工作框架，待新聊天推进方案

## 目录结构

- `00_项目主页`：项目定位、新聊天启动提示、总体判断。
- `01_既有系统与专利基础`：既有 VTE 智能管理系统、专利、软著、论文和前期成果索引。
- `02_产品定位与应用场景`：智能体定位、用户角色、临床流程节点和应用边界。
- `03_智能体架构与工作流`：智能体角色、任务流、工具调用、人机协同和审计机制。
- `04_RAG知识库与规则库`：指南、院内路径、风险评分、药物规则、患者教育材料。
- `05_数据接口与系统集成`：HIS/EMR/LIS/PACS/护理/随访等接口和部署边界。
- `06_临床验证与评价`：回顾性验证、前瞻性试点、真实世界评价、医护采纳与预警疲劳。
- `07_合规安全与责任边界`：数据安全、伦理、隐私、自动化边界、医疗器械属性判断。
- `08_横向课题与合作研发`：合作方沟通、横向课题、任务书和研发里程碑。
- `09_知识产权与成果转化`：专利延展、软著、产品化、注册和商业合作边界。
- `10_版本记录与归档`：版本记录、定稿和历史材料。

## 当前判断

VTE 智能体不是“VTE 系统 + 聊天窗口”。它应是围绕 VTE 风险评估、动态预警、防控建议、复评提醒、随访教育和质控管理的临床任务执行体。核心价值在于把已有系统能力、专利基础、指南知识、院内流程和多角色协同串成可审计、可验证、可持续迭代的工作流。
