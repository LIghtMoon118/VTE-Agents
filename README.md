# VTE 智能体 MVP 研发交接包

生成日期：2026-07-11  
交接对象：研发团队、院内信息/数据团队、合作工程团队  
项目定位：VTE 智能预警与全生命周期管理主线下的一期专病智能体 MVP

## 1. 这个包交付什么

本包不是单纯的演示页面归档，而是把当前 VTE 研究主线、智能体产品设计、MVP 源码、接口字段、知识库、临床边界和下一阶段研发任务整理成一个可继续推进的交接包。

核心内容：

- `01_交接文档/`：给研发团队阅读的总说明、运行验收、工程化任务和风险边界。
- `02_MVP源码/vte-agent-mvp/`：当前可运行的一期 MVP 源码副本。
- `03_项目资料/`：项目主页、产品定位、架构、RAG/知识图谱、接口字段、临床验证、合规和知识产权资料。

## 3. 当前 MVP 的一句话判断

当前 MVP 已经具备本地运行、病例文本分析、PDF/Word/图片导入预处理、规则/RAG 输出、病历证据链审查、路径质控、知识图谱、模型适配层和接口草案；但它仍是一期原型，不是已完成临床验证的软件产品。

研发团队下一步不应只做界面美化，而应优先完成四件事：

1. 把病例输入、病历抽取、结构化字段和分析结果做成稳定后端服务。
2. 把当前规则/RAG 替换为可审核、可版本管理、可追溯的知识库与推理链。
3. 完成院内数据字段映射、权限、审计、脱敏和测试环境部署。
4. 用脱敏病例建立临床验证集，逐项验证抽取准确性、风险识别、建议一致性和病历质控价值。

## 4. 数据与合规提醒

当前包内不应放入真实患者隐私数据。研发团队如需调试真实病历，必须在院内授权环境内处理，不能将真实病历上传到外部模型或公网环境。

智能体输出定位为临床辅助、病历完整性提示和流程质控支持，不作为自动诊断、自动治疗或自动开立医嘱依据。
